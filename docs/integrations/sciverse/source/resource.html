resource 按相对路径下载附件

用相对路径 file_name 下载论文图片和其他二进制附件。

概述

resource 接口用于拉取论文插图、实验图、解析图等文献相关二进制附件。file_name 通常来自检索结果、解析结果或正文中的图片路径，只传相对路径，不要传完整 URL；响应为二进制流，带 Content-Type 与 Content-Disposition。

不适用场景

  • · 不用于搜索图片或发现论文，请先用 agentic-search、meta-search 或 content 找到资源路径。
  • · 不用于解释图片内容，上层多模态模型负责理解。
  • · 不用于读取正文文本，请使用 content。

能力边界

  • · resource 只接受安全的相对 file_name，不接受完整 URL、绝对路径、.. 或反斜杠。
  • · 返回通常是 image/png、image/jpeg、application/pdf 等二进制内容。
  • · 资源是否存在取决于目标文献解析结果和账号权限。
  • · 调用方应校验 content-type 和响应状态，再决定展示或下载。

鉴权

统一使用 API Key Bearer Token 鉴权，详见统一鉴权章节。在 HTTP Header 中加入：

Authorization: Bearer YOUR_API_TOKEN

请求示例

curl -G https://api.sciverse.space/resource \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  --data-urlencode "file_name=papers/2025/abcd/fig1.png" \
  -o fig1.png

请求参数（URL query）

字段类型必填说明
file_namestring必填资源相对路径；不得包含 \、..，且不得以 / 开头。

响应结构

响应为二进制流；常见 Content-Type：image/jpeg、image/png、application/pdf 等。

字段类型说明
Content-Typeheader附件的 MIME 类型。
Content-Dispositionheader文件名与下载提示。
X-Request-IDheader请求追踪 ID。
bodybinary附件原始二进制内容。

  • · 不应重试：400 / 401

FAQ

file_name 从哪里获取？

通常来自 content Markdown、解析结果、agentic-search 或 meta-search 返回的资源引用字段。

可以传完整图片 URL 吗？

不可以。resource 只接受平台返回的相对路径，避免 SSRF 和路径穿越风险。

resource 会解释 Figure 或 Table 吗？

不会。它只返回二进制资源；图表理解需要上层多模态模型或业务逻辑。

下载失败怎么处理？

检查 file_name 是否来自 Sciverse 返回、路径是否安全、账号是否有权限，并根据 404/403/429 做降级或重试。

还没有 API Key？

登录控制台「密钥」即可创建。同一套 API Key 可用于已开通的 Sciverse、点石与 Skills 能力，提供基础试用额度，具体以账号权限为准。
